FAQ

Frequently Asked Questions

How does the AI match score work? ▼
The AI engine uses a 4-factor scoring model: technical skills overlap (40%), preferred domain match (25%), CGPA vs minimum requirement (20%), and areas of interest alignment (15%). These combine for a score out of 100.
Can I add students without logging in? ▼
Yes — the Student Management page is accessible to anyone. When connected to the backend, admin operations like deleting records require an admin login via the Admin Dashboard.
Where is student data stored? ▼
Frontend mode uses browser LocalStorage for offline demonstration. When the Node.js/MongoDB backend is running, data is stored in a MongoDB database and accessed via REST API.
How do I set up the backend API? ▼
Install Node.js and MongoDB, copy the backend folder, run npm install, configure the .env file with your MongoDB URI, run npm run seed, then npm run dev. See README.md for full instructions.
Can companies register on the system? ▼
In the current version, internship listings are managed by the admin. Company self-registration is planned as a future enhancement in the next version of the project.
Is this project suitable for viva presentation? ▼
Absolutely. The system demonstrates AI/ML concepts, full-stack web development, database design, REST API architecture, and modern UI/UX design — all core topics examiners look for in a final year capstone project.
